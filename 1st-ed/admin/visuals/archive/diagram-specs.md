# Diagram Specifications

*The Remote Worker’s Survival Guide in the AI Age*
Author: Gary G. Bayes, BABA, MBA

Status: Working Document
Purpose: Define the exact structure and text used inside every diagram in the book.

This document ensures:

* diagrams remain consistent with the manuscript
* visual revisions remain controlled
* diagrams can be recreated quickly from templates
* diagram text remains synchronized across chapters

Each diagram corresponds to an entry in:

```
00-admin/visual-index.md
```

Diagram source files are stored in:

```
03-visuals/chapter-xx/
```

# Diagram Maintenance Rules

When editing diagrams:

1. Update this file first.
2. Modify the diagram using the draw.io template.
3. Export updated SVG.
4. Replace the visual in `03-visuals/`.
5. Confirm the visual index remains accurate.

---

# Version Control Note

If diagram content changes significantly, include the update in commit messages.

Example:

```
docs: update AI productivity stack diagram text
```

This ensures visual logic changes remain traceable in Git history.
