# .visual-map.md

Visual Architecture Map

Project: **The Remote Worker’s Survival Guide in the AI Age**
Author: Gary G. Bayes, BABA, MBA

Purpose:
This document defines the **visual system used throughout the book**.
It ensures that diagrams, infographics, screenshots, and mockups are used consistently and appear in the correct chapters.

This file is intended for:

• authors
• editors
• AI editing assistants
• layout tools

---

# Visual Philosophy

Visuals exist to:

• clarify complex ideas
• reinforce key frameworks
• break up dense text
• make workflows easier to understand

Each visual should support a **specific concept**, not simply decorate the page.

Most chapters contain **1–3 visuals**.

# Visual Integration Rules

Each visual must follow this structure inside chapters:

Concept explanation
Lead-in sentence

Visual

Caption

Interpretation paragraph

Example Markdown:

![Prompt Structure Framework](../../visuals/core-theme/prompt-structure-framework.svg)

*Figure 3.1 — Prompt Structure Framework*

This framework shows how structured prompts combine role, context, and instructions to produce more reliable AI responses.

---

# Caption Format

Captions follow this format:

Figure X.X — Title

Where:

X.X = chapter number + figure number

Example:

Figure 5.2 — Research Workflow Using AI

---

# Visual Density Guidelines

Recommended density:

• minimum: 1 visual per chapter
• typical: 2 visuals per chapter
• maximum: 3 visuals per chapter

Avoid adding visuals unless they improve comprehension.

---

# File Naming Rules

Visual files should use descriptive names:

Examples:

research-workflow-diagram.svg
automation-workflow-diagram.svg
ai-productivity-stack.svg

Avoid:

image1.svg
diagram-final.svg

---

# Maintenance Rules

When adding a new visual:

1. Add the file to the correct directory.
2. Add an entry to this visual map.
3. Update the chapter where the visual appears.
4. Ensure the caption follows the standard format.
